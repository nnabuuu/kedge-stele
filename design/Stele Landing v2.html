决策会蒸发。
实录把它刻下来。

你做的每个决定，背后有它的理由、放弃掉的选项，还有一堆先搁着没定的事。过几天，这些常常只剩一个结论。实录在你决定的当下就把它们一起刻下来，日后落在任何一件事上，都能回看它当初是怎么定的。

视角

原子是决策，不是报告

一个 feature 是一串决策串起来的。推迟做某件事本身是一个决策；某件事还悬着，是一个待决的决策。底层存的就是这样一张决策的图，节点是决策，边是它们之间的关系。报告、待办、回顾，全都是这张图的投影。

视图都是当场从图里查的。图一变，旧报告自己就跟着变了。

每天在用的两件事

把脑子里悬着的事，外化成图

什么在等我

所有未闭合的回路一次倒出来，open 的问题、还没被解决的 deferred。可能到期的（触发条件刚满足）该复查的排在前面。悬着的事不用靠脑子记，都落在这张表上。

这件事怎么发生的

指着一个文件、一个决定，顺着图回看它的来历，谁提的、当时什么约束、否决了什么、为什么这么选。入口是这件事本身，不用从头把时间线翻一遍。

用法

一个本地 MCP server，四个工具

Node ≥ 22.6 · 连接 Claude Code · stdio 传输
stele · ~/.provenance/decisions.db
decision_capture

记一个决策。agent 从对话里起草全字段，你只确认；落库后返回它提议的关联边。

decision_resume

「什么在等我」，所有未闭合回路，可能到期的在前。

decision_trace

「怎么发生的」，按决策 id 看图邻域，或按 entity 看所有碰过它的决策。

decision_resolve

连一条边。resolves 把目标翻成已解决，这就是那个跨 session 的缝合。

02

讨论完一个设计，敲 /decision，agent 起草，你确认。

03

下次回来先 decision_resume，一眼看清有哪些开放回路。
